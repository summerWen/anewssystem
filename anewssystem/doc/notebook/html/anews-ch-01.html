第 1 章 技术选型

1.1. 选择哪些工具包做整合

  1. spring

    我们需要它的pi依赖注入作为胶水，把其他工具包粘在一起

    还需要它的aop，进行事务配置，权限管理

    mvc使用springmvc，因为extjs使得mvc这一层变得不必要了。

  2. hibernate

    作为ORM方案，使用hibernate3对JPA的支持，使用annotation简化配置。

  3. acegisecurity

    与spring结合，控制权限

  4. extjs

    提供js控件，实现富客户端

    extjs与服务器之间交互采用json格式传递数据，目前采用的json-lib-2.0在多线程环境下会有同步问题，2.1-snapshot已经解决了这个问题，但正式版尚未发布。json-lib在处理循环引用的时候也有很大的问题，处理hibernate双向关联的时候100%出错。

    考虑自己写一个json实现，参考dwr中生成s1={};s2={};s1.s2=s2;的方式。

  5. dwr

    使用dwr和 commmons-fileupload制作上传文件的进度条。

    考虑使用dwr-2的服务器推，制作在线聊天。

1.2. 待选方案

（以后可能用上，也可能用不上的工具包）：

  1. Web Service

    CXF是由Objectweb Celtix和Codehaus XFire合并成立的。与axis1~2比较，更易于整合到spring中。如果以后需要用到Web Service的话，它一定是首选。

  2. JMS

    activeMQ，根据springside的推荐，使用activeMQ作为jms的实现方式。

  3. 全文搜索

    compass同另外一个hibernate-search都是基于 lucence，对数据库的全文搜索引擎。

    之前使用过compass，在单元测试的时候会出现内存溢出的问题。

  4. 定时调度

    quartz，被springside称为唯一免费的选择。

  5. 报表

    JasperReport负责文本报表，JFreeChat图形报表。

  6. 工作流

    jbpm，目前仅用过的工作流引擎。osworkflow，号称最灵活的工作流引擎。两者都支持hibernate。

1.3. 运行环境

  1. jdk5

    支持范型generic和注释annotation，可大大减少配置提高抽象程度。

    考虑用Retrotranslator使项目可以在JDK1.4上运行

  2. servlet容器：tomcat-5.5

    应用甚广的服务器，基本用到jsp的人都会有。

    maven2可以使用 jetty-6作为插件进行运行和测试，单独使用jetty的人不多。

  3. 数据库

    采用hsqldb作为嵌入式数据库，可与工程绑定发布，用户下载体验版后不需要额外安装数据库，减少了配置出错的机会。

    hibernate本身支持多数据库，包括mysql,sqlserver,oracle等，未来替换成企业数据库也不是问题。

  4. 数据部署

    DBDeploy让数据库脚本也可以实现版本控制。

1.4. 测试

  1. 单元测试

    junit。经典单元测试工具，基本所有工具都支持。

    js的单元测试工具 jsunit不知如何应用。

  2. mock

    easymock使用烦琐，但有的时候还必须依靠它制作mock进行测试。更多时候使用spring-mock对servlet进行测试，spring-mock还提供了几个常用的测试基类，可以缓存xml配置，支持事务。

  3. 测试覆盖率

    cobertura，免费的测试覆盖率检测工具，1.9版本已经慢慢追上clover的效果了，从maven2的角度来看，配置比clover更简便，而且不需要像clover那样每月去申请授权。

  4. 集成测试

    selenium被众口称赞的集成测试工具，不过想灵活应用还需要一定的时间。

    同类产品还有httpunit和jwebunit。

    使用maven2做集成测试时，可以搭配cargo控制服务器的启动与停止。

  5. 压力测试

    jmeter听过但是没用过。

1.5. 项目管理

使用maven2进行项目管理，配置了jalopy对java代码进行美化，支持多种插件，进行自动打包，测试，代码分析，生成报表。

  1. checkstyle进行源代码格式分析。

  2. pmd, cpd检查重复代码。

  3. findbugs检查程序漏洞。
